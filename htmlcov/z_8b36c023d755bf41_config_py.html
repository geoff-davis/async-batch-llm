15class RetryConfig: 

16 """Configuration for retry behavior. 

17 

18 ``max_attempts`` bounds retries for *content/transport* failures (validation 

19 errors, timeouts, connection errors, generic 5xx). **Rate-limit errors are 

20 exempt from this budget**: a 429 / quota / coordinated-cooldown signal is a 

21 "wait and try again", not a failed attempt, so it does not consume an 

22 ``attempt`` — the same logical ``attempt`` number is retried after the 

23 cooldown. Rate-limit retries are instead bounded separately by 

24 ``max_rate_limit_retries`` (a backstop against an endpoint that throttles 

25 forever). This keeps model-escalation strategies (which key behavior off the 

26 attempt number) from escalating just because the endpoint was busy. 

27 """

35 # consuming its max_attempts budget. 0 = rate limits immediately count as 

36 # failures. Exceeding it fails the item with a clear "exceeded N rate-limit 

37 # retries" error.
